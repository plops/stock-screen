# S&P 500 Macro Alignment Stock Screener and Backtester

This project implements a stock screening and backtesting framework based on Federal Reserve policy (interest rates & liquidity). The market is divided into four quadrants based on monetary policy regimes, and stocks are classified into four corresponding investment profiles (A, B, C, D) using financial metrics (revenue growth, earnings growth, forward P/E, and debt-to-EBITDA).

The system will download macro data (FED balance sheet & interest rates) from FRED, S&P 500 stocks from Wikipedia, and corporate fundamentals/historical prices from Yahoo Finance. All data is cached in a local SQLite database to prevent redundant API calls.

## User Review Required

> [!IMPORTANT]
> **Automatic Virtual Environment Execution (PEP 723)**:
> We will use `uv run <script>` with PEP 723 inline script metadata. This ensures all required python libraries (`pandas`, `yfinance`, `requests`, `matplotlib`, `jinja2`) are automatically installed in an isolated, temporary environment without modifying your global Python setup.

> [!NOTE]
> **API Keys**:
> We will fetch Federal Reserve data (WALCL, FEDFUNDS) directly via FRED's public CSV export URLs (`https://fred.stlouisfed.org/graph/fredgraph.csv?id=...`). This avoids the need for a FRED API key, making the tool instantly usable for anyone.

> [!WARNING]
> **yfinance Fetching Speed and Rate Limits**:
> Fetching detailed fundamental data for all 500 S&P 500 companies can take 5–10 minutes due to rate-limiting by Yahoo Finance. We will:
> 1. Implement robust SQLite caching: data will only be fetched once per day per ticker.
> 2. Include a progress bar and graceful interruption (saving partial progress so a resumed run doesn't restart from scratch).
> 3. Provide a `--limit` flag to test with a small batch (e.g. 20 stocks) before doing a full run.

### Core Scripts

#### [NEW] [stock_screen.py](file:///home/kiel/stage/stock-screen/stock_screen.py)
This script is the main entry point for current stock screening. It:
- Scrapes the S&P 500 tickers from Wikipedia.
- Downloads macro data (WALCL, FEDFUNDS) from FRED.
- Determines the current macro regime (QE vs. QT, High vs. Low interest rate).
- Downloads current fundamentals (P/E, Debt/EBITDA, Revenue & EPS Growth) from Yahoo Finance using `yfinance`.
- Performs fallback calculations (e.g., `debtToEbitda` calculated as `totalDebt / ebitda` if the primary field is missing).
- Caches all data in an SQLite database `stock_data.db`.
- Classifies stocks into the 4 profiles based on relative market medians (to adapt to changing environments).
- Outputs `screener_results.csv` and `report.md`.

#### [NEW] [backtest.py](file:///home/kiel/stage/stock-screen/backtest.py)
This script performs historical backtesting of the theory. It:
- Downloads historical Fed data since 2002 (when weekly WALCL data begins).
- Segments history into Q1, Q2, Q3, and Q4 regimes.
- Classifies stocks historically or simulates portfolio performance of the current portfolios during historical regimes.
- Downloads historical price data for the S&P 500 tickers.
- Simulates the performance of holding Defensive (Q4), Speculative (Q1), Value (Q3), and Moderate (Q2) stock cohorts.
- Generates comparative performance charts.

#### [NEW] [README.md](file:///home/kiel/stage/stock-screen/README.md)
Documentation file explaining how to run the screener, refresh the data, and execute the backtests.
